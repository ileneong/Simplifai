# Simplifai — AI Tool Source List

The watch list for Simplifai's weekly "What's New in AI" research. Two boundaries set the scope:

- **Whole-business (the task):** any AI tool that helps an owner *run their business* — marketing, yes, but also customer service, sales, operations, admin, scheduling, finance, and everyday productivity.
- **Wellness-led (the audience):** Simplifai's home turf is health and wellness owners — nutritionists, dietitians, therapists, coaches, clinicians, gyms, studios, clinics, spas, and supplement/health-food/telehealth/wellness-tech founders. The tools below are mostly generic; judge each *update* first by "would this change how a **wellness** owner works this week?" — but a clearly useful update for any small owner still qualifies. Wellness is the lead lens, not a hard filter.

If an update wouldn't change how a non-technical small owner (wellness first) works this week, it doesn't belong here.

This is the one file to keep current as the toolscape shifts. Add a source when a tool earns a place in the Simplifai Stack; retire one when it stops mattering to owners.

**How the research skill uses this:** [`.claude/skills/simplifai-trend-research.md`](../../.claude/skills/simplifai-trend-research.md) sweeps every source below each week, keeps only verified shipped updates, and translates them into the weekly digest.

## Maintenance
- **Owner-value filter:** before adding a source, ask "would a busy small owner — wellness first — with no technical background care about this tool's updates, for *any* part of running their business?" If no, leave it out.
- **Wellness-led lens:** the tools are generic, the audience leans wellness. When an update could become a post, frame it for a wellness owner by default (a clinic's bookings, a coach's client comms, a supplement brand's CRM) and pass the regulated-claim flag through anything touching health outcomes, supplements, or mental health. Non-wellness framings are fine when the update is broadly useful.
- **Primary-source rule:** aggregators are leads only. Every claim that reaches a Simplifai post must be traced to and cited from the tool's own announcement.
- **Stay accessible:** keep the list to tools a non-technical solo owner can actually adopt. Drop developer/API-only and enterprise-gated tooling.
- Review this list quarterly alongside the Simplifai Stack.
